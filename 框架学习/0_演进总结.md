# 排序模型演进路线

## 一句话理解每个模型的核心

```
Wide&Deep  → "我用线性模型记住了历史，用DNN泛化新情况"
DeepFM     → "Wide那部分手动设计交叉太累了，用FM自动学二阶交叉"
DCN        → "FM只能二阶太弱了，我用Cross Network显式学任意阶交叉"
AutoInt    → "DCN的交叉是bit-wise太粗糙，我用Self-Attention做vector-wise动态交叉"
```

## 演进关系图

```
Wide&Deep (2016)
│   Wide: 手动交叉特征（线性）
│   Deep: 隐式泛化
│   问题: Wide需要人工feature engineering
│
├──→ DeepFM (2017)
│       FM: 自动二阶交叉（替代Wide）
│       Deep: 隐式泛化
│       问题: 只有二阶，高阶靠Deep凑
│
├──→ DCN (2017)
│       Cross: 显式高阶交叉（层数=阶数）
│       Deep: 隐式泛化
│       问题: bit-wise交叉，不够灵活
│
└──→ AutoInt (2019)
        Self-Attention: 动态vector-wise交叉
        Deep: 隐式泛化
        问题: 计算量大，O(n^2)
```

## 关键对比表

| 模型 | 交叉方式 | 交叉阶数 | 是否手动 | 交叉粒度 |
|------|---------|---------|---------|---------|
| Wide&Deep | Wide线性 | 1阶（需手动设计高阶） | 是 | - |
| DeepFM | FM分解 | 2阶 | 否 | vector-wise |
| DCN | Cross Network | 任意阶(k层=k阶) | 否 | bit-wise |
| AutoInt | Self-Attention | 任意阶(k层=k阶) | 否 | vector-wise + 动态权重 |

## 什么是 bit-wise vs vector-wise？

```
假设特征A的embedding = [a1, a2, a3]
假设特征B的embedding = [b1, b2, b3]

bit-wise（DCN）：每个维度独立交叉
  交叉 = [a1*b1*w1, a2*b2*w2, a3*b3*w3]  权重w1,w2,w3是共享的

vector-wise（AutoInt/DeepFM）：整个向量参与交叉
  交叉 = f([a1,a2,a3], [b1,b2,b3])  注意力权重是动态计算的
```

## 面试常见问题

**Q: 为什么不用纯DNN？**
A: 纯DNN学特征交叉是隐式的，效率低。显式交叉模块（FM/Cross/Attention）更快收敛，效果更好。

**Q: 实际工业界用哪个？**
A: DCN用得最多（Google系），DeepFM也很多。AutoInt效果好但计算贵。实际中常把这些模块组合使用。

**Q: 这些模型的Deep部分有区别吗？**
A: 几乎没区别，都是MLP。核心差异全在交叉模块上。
