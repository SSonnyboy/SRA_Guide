# 评估指标

## 1. 混淆矩阵
         预测正类   预测负类
真实正类   TP         FN
真实负类   FP         TN

## 2. 准确率
- acc = (TP+TN) / (TP+FP+TN+FN) 
- 当正负样本极度不平衡时 模型全部预测为负都有99的准确率 但是业务上没有意义

## 3. 精确率
- precision = TP / (TP+FP)
- 预测的正类中哪些时正确预测的

## 4. 召回率
- Recall = TP / (TP+FN)
- 真实为正样本中多少被找出来预测为真

## 5. F1 Score
- 精确率和召回率的调和平均
- F1-score = 2*(Precision*Recall) / (Precision + Recall)

## 6. AUC & GAUC 衡量排序区分能力
- AUC：从所有正样本中随机抽取一个 再从所有负样本中随机抽取一个 模型对这个正样本的预测打分大于负样本的预测打分概率
- AUC = 1.sort 2.(sum(rank.pos) - M(M+1)/2)/(M*N) 
- ROC 需要一个二维图来表示不同阈值下的混淆矩阵表现 所以有ROC曲线

- 推荐不是二分类，而是"把用户可能喜欢的排在前面" AUC衡量的是正样本排在负样本前面的概率，直接反映排序能力 
- ACC依赖固定的0.5分类阈值，而推荐中不同业务场景需要不同的截断点
- 忽略用户差异性 不同用户的兴趣尺度不同，全局AUC会被活跃用户主导（贡献更多样本）
- GAUC的核心思想：先算每个用户的AUC，再按权重平均。
- GAUC = 曝光数加权（活跃用户共享）、点击数加权（高价值用户）、均匀加权

## NDCG： 归一化折损累计收益：衡量排序质量
- 含义：衡量相关性等级的排序结果 及其关注排在什么位置
- CG：前K个推荐结果的相关性之和 ：[0, 0, 3] [3, 0, 0] 两者相同
- DCG：加入位置惩罚： DCG = sum(rel/log(i+1))
- IDCG: 理想最优DCG： IDCG = sum(sortec(rel)/log(i+1))
- NDCG: 归一化折损累计收益：NDCG = DCG、IDCG

## MRR 平均倒数排名
- 只关注第一个相关物品出现在什么位置
- MRR = i/U sum(1/rank)  对于所有用户统计平均

## Hit Rate 命中率
- 在推荐TopK列表中 只要包含用户真正感兴趣的物品 就算做命中 所有用户hit求平均
