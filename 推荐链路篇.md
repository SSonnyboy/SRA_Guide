# 推荐链路篇

## 1. 链路概览包括：
- 召回层：从海量物品库中快速筛选用户可能感兴趣的一小部分候选物。需要快和高召回
- 粗排层：比召回精准比精排更轻量 通常是双塔模型或小DNN 可能会做截断
- 精排层：对粗排传入候选集打分 使用复杂深度学习模型和交叉特征模型
- 重排层：结合业务规则 多样性 打散 去重 提权 

## 2. 召回的认识：
- 经典模型 ItemCF、双塔模型、图召回
    - ItemCF：遍历U2I 两层循环 物品i，j的sim+=1
        - 缺点权重单一：
            - 考虑惩罚高活用户：权重 = 1/log(1+n) n=用户的交互数
            - 考虑共现时间衰减：共现时间越短越相似 α*decay^|i-j|-1
            - 考虑对热度物品惩罚：w = w /cnt(i)*cnt(j)
    - Swing: ItemCF的优化版本，考虑惩罚小圈子 鼓励共现用户的多样
        - 获取共现字典{(i, j):[u1, u2, ..]} 以及U2U
        - 遍历U2I 两层循环 物品i，j 遍历共现字典：w += 1/(α+ val.U2U)
    - userCF：遍历I2U 两层循环 用户i，j的sim+=1
    - 双塔模型：用户/物品特征embedding+concat+NN->vector
        - 损失优化：
            - pointwise：sigmoid-BCE损失 = loss_bce = -ylogp-(1-y)log(1-p)
            - pairwise：BPR损失 =  loss_bpr = -log(sigmoid(score_pos-score_neg))
            - listwise：softmax-CE损失 = sum(-ylogp).mean
        - 推理：用户实时计算、物品存faiss 近似最近邻查找
    - 图召回：传统的序列模型+双塔难以捕获用户+物品的高阶关系，GNN可以邻居聚合
        - 如何归一化邻接矩阵？A = D\*A\*D   D = D^(-1/2) 度矩阵
- 召回层-正负样本构造：
    - 正样本：点击、点赞、收藏、转发、下单
    - 负样本构造：
        - 简单负样本：全局随机采样 batch内负样本
        - 困难负样本：召回但被排序截断的
    - 为什么召回层中不选择曝光未点击的作为负样本？
        - 召回和排序是不同的优化目标：召回关注用户可能对什么物品感兴趣、排序关注用户对哪些物品更感兴趣。
        曝光未点击物品已经是召回和精排筛选出相对优质物品，作为负样本可能会破坏召回的粗筛访华能力


## 3. 排序的认识：
- 基础经典方法：wide&deep、DeepFM、DCN、AutoInt
    - wide&deep：
        - Wide：线性模型，负责记忆—— 记住历史中共现的特征组合，比如"买过尿布的人也买了啤酒"
        - Deep：DNN，负责泛化—— 学到没见过的特征组合，通过embedding的低维稠密表示
        - 缺点：
            - Wide 部分的交叉特征需要人工手动设计（feature engineering）
            - Wide 和 Deep 是分开的，没有真正的特征交叉
            - 组合空间爆炸 稀疏到爆炸
    - DeepFM:
        - 用 FM（因子分解机）替换 Wide&Deep 的 Wide 部分
        - 缺点：
            - FM 只能做二阶交叉（两两组合），更高阶的交叉靠 Deep 凑
            - Deep 部分的高阶交叉是隐式的，不可解释
    - DCN：
        - 核心公式：x_{l+1} = x_0 * (w^T * x_l) + b + x_l 即：用 x0 乘以 xl 的线性变换，实现显式交叉
        - 缺点：w^T × x_l 是一个标量（一个数）！ bit-wise 交叉
    - AutoInt：
        - 核心思想：用 Multi-Head Self-Attention 自动学习特征交叉
        - 把每个特征的 embedding 当作一个 token
        - 用 Transformer 的 Self-Attention 让特征之间互相"看"
        - 多头注意力 = 同时从多个角度学交叉关系
        - 缺点：过拟合风险更高（参数更多）

- 特征交互优化：SENet、PPNet
- 召回层-正负样本构造：
    - 正样本：点击、点赞、收藏、转发、下单
    - 负样本构造：
        - 简单负样本：全局随机采样 batch内负样本
        - 困难负样本：召回但被排序截断的 曝光未点击

## 4. 重排的认识：
- 重排决定了用户最终看到的视觉呈现，兼顾商业利益+用户体验
- MMR平衡相关性和多样性：Score = λ * Sim(u, i) - (1-λ) * max sim(i, j) 贪心选择
- 滑动窗口打散机制：约束同作者 同类目
- 重排去偏：
    - 位置去偏：用户更倾向于点击排名靠前的物品，即使靠后的物品更相关，导致训练数据"位置即标签"的假象。
    - 流行度去偏：流行度偏差指热门物品因曝光机会多而获得更高点击，模型倾向于推荐热门物品，造成"富者愈富"的马太效应。
